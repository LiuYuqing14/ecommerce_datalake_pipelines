# Validation Reports

This directory contains auto-generated quality reports from the pipeline's three validation gates.

## Report Types

### 1. Bronze Quality (`BRONZE_QUALITY_*.md`)
- **Generated by:** `src.validation.bronze_quality`
- **When:** Before Base Silver transformation
- **Checks:**
  - Manifest existence and validity
  - Row count reconciliation
  - Schema drift (unexpected columns)
  - Partition coverage

### 2. Dimension Snapshot Quality (`DIMS_SNAPSHOT_QUALITY_*.md`)
- **Generated by:** `src.validation.dims_snapshot`
- **When:** After dimension refresh, before Base Silver
- **Checks:**
  - Primary key integrity (null/duplicates)
  - Schema consistency
  - Snapshot freshness vs ingest_dt

### 3. Silver Quality (`SILVER_QUALITY_*.md`)
- **Generated by:** `src.validation.silver`
- **When:** After Base Silver transformation
- **Checks:**
  - Row loss (Bronze vs Silver)
  - Quarantine rates and reasons
  - Primary key uniqueness
  - Foreign key integrity

### 4. Enriched Quality (`ENRICHED_QUALITY_*.md`)
- **Generated by:** `src.validation.enriched`
- **When:** After Enriched Silver transformation
- **Checks:**
  - Business rules (semantic checks)
  - Numeric sanity checks (non-negative)
  - Schema consistency
  - Minimum row thresholds

## Usage

Reports are generated automatically by the pipeline. To generate manually:

```bash
# Bronze
ecomlake bronze validate --partition-date 2025-10-15

# Silver
ecomlake silver validate --partition-date 2025-10-15

# Enriched
ecomlake enriched validate --ingest-dt 2025-10-15
```

## Storage

- **Local:** `docs/validation_reports/`
- **Production:** `gs://$REPORTS_BUCKET/validation_reports/<run_id>/`
